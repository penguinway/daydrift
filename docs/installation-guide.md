# 安装使用说明

本文说明如何安装「时光 · DayDrift」、启动可选的实时状态服务，并完成首次使用配置。

## 适用范围

- 客户端：Android 手机或 Android 模拟器。
- 基础功能：纪念日、提醒、心愿单均可离线使用。
- 「TA在干嘛」功能：需要一台可被双方手机访问的 Node.js 服务端。

## 1. 安装客户端

### 从 Release APK 安装

1. 在 GitHub Release 下载 APK。
2. 现代 Android 手机优先选择 `app-arm64-v8a-release.apk`；旧设备可尝试 `app-armeabi-v7a-release.apk`。
3. 将 APK 传到手机并打开安装。
4. 如果系统提示“未知来源应用”，按系统提示允许本次安装。

### 从源码构建安装

本机需要 Flutter 3.41.4+、JDK 17+、Android Studio / Android SDK。

```bash
flutter pub get
flutter run
```

构建发布 APK：

```bash
flutter build apk --release --split-per-abi
```

生成文件位于 `build/app/outputs/flutter-apk/`。

## 2. 启动实时状态服务（可选）

只有使用「TA在干嘛」时才需要服务端。

```bash
cd server
npm install
npm start
```

服务默认监听 `0.0.0.0:3000`。确认服务可用：

```bash
curl http://localhost:3000/api/health
```

### 登记配对关系

服务端不会自动配对用户，需要管理员先登记双方 ID：

```bash
curl -X POST http://localhost:3000/api/admin/pair \
  -H "Content-Type: application/json" \
  -d '{"user1":"alice","user2":"bob"}'
```

查看已有配对：

```bash
curl http://localhost:3000/api/admin/pairs
```

生产部署建议使用 PM2、systemd 或容器托管，并在公网域名或局域网固定 IP 后配置 HTTPS / 反向代理。当前服务端配对和状态数据保存在内存中，重启会丢失。

## 3. 首次配置 App

1. 打开 App，底部有「时光」「心愿」「TA」三个标签。
2. 若使用模拟器连接本机服务端，默认地址 `http://10.0.2.2:3000` 通常可直接使用。
3. 若使用真机，进入「TA」页右上角设置，将服务器地址改为手机可访问的地址，例如 `http://192.168.1.20:3000` 或线上域名。
4. 在「TA」页输入服务端登记过的用户 ID，例如 `alice`。
5. 系统提示使用情况访问权限时，跳转到设置页，为「时光」开启权限。
6. Android 13+ 若使用纪念日提醒，还需要允许通知权限。

## 4. 使用纪念日

1. 进入「时光」页，点击右下角 `+`。
2. 填写事件名称，选择日期。
3. 如需提醒，打开「纪念日提醒」，选择当天或提前 1 / 3 / 7 天，并设置提醒时间。
4. 点击保存。
5. 在列表中可编辑或删除事件；首页会显示已过去总天数和年月日拆分。

## 5. 使用心愿单

1. 进入「心愿」页，点击右下角 `+`。
2. 填写标题、备注、分类和可选目标日期。
3. 分类支持内置选项，也可点击「+ 自定义」新增。
4. 保存后在心愿卡片上切换状态：`想做 → 计划中 → 已完成`。
5. 已完成心愿会归入折叠区域，可展开查看。

## 6. 使用「TA在干嘛」

1. 确认服务端已启动，并已登记双方配对。
2. 双方 App 都配置同一个服务端地址。
3. 双方分别用自己的用户 ID 登录。
4. 开启使用情况访问权限后，App 会定期上报当前前台应用。
5. 对方页面会通过 WebSocket 实时显示当前 App 名称、包名和更新时间。

## 常见问题

### 登录失败

- 检查用户 ID 是否已通过 `/api/admin/pair` 登记。
- 检查手机能否访问服务端地址。
- 真机不能使用 `10.0.2.2` 访问电脑；请改用电脑局域网 IP 或公网域名。

### TA 状态不更新

- 确认双方都已登录同一服务端。
- 确认 Android “使用情况访问权限”已开启。
- 检查服务端 `/api/health` 是否正常。
- 检查网络是否阻止 WebSocket 连接。

### 纪念日提醒不响

- 确认事件已开启提醒。
- Android 13+ 需要通知权限。
- 部分系统会限制后台任务，必要时在系统电池设置中允许 App 后台运行。

### 服务端重启后配对丢失

当前 `server/server.js` 使用内存保存配对和状态。重启后需要重新调用 `/api/admin/pair` 登记；生产使用建议接入 Redis、SQLite 或其他持久化存储。
